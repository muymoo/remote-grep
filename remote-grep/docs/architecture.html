Log file Testing Strategy

The design will include 50 test log files on each node included in the distributed system, each with 100 lines of text. This means 3 nodes with 5000 lines of text total.


Figure: Diagram of the template files to be created.

To give a range of dates within the text data, each file will only have dates and times for a single day. As the files will be numbered from [0-49] inclusively, each file will have timestamps from the day March 1, 2013 + days.

Single machine queries (still not sure if these will work):

  • grep common (e.g. grep -rni linux6)
Multiple machine queries:
  • grep common (e.g. grep -rni WARNING) - should pull 9000 lines
  • grep common (e.g. grep -rni 2013) - should pull all lines
  • grep rare (e.g. grep -rni linux6) - should pull 5000 lines
